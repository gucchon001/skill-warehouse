# コード品質レビュアー用チェックリスト（第2段）

サブエージェント `code-quality-reviewer` 用。**第1段が PASS のあとだけ**起動する。仕様の網羅は再チェックしない（逸脱していたら第1段に戻す）。

## 入力の前提

- 対象: 同じ差分・ファイル。
- チームのコーディング規約スキルや lint 設定があれば Read してからレビュー。

## 検証項目

1. **可読性**: 命名、関数の長さ、コメントの要否、不要複雑さ。
2. **一貫性**: 既存パターン（エラー処理、ログ、DI）との揃い。
3. **パフォーマンス・複雑度**: 明らかな N+1、不要なループ、過大なメモリ（コンテキストに応じて）。
4. **テスト**: 変更に応じたテストの有無・ギャップ（方針はプロジェクト依存）。
5. **型安全・契約**: TypeScript 等での型、境界での検証の妥当性。

## 出力形式（例）

- **判定**: PASS / FAIL / SUGGESTIONS（軽微のみ）
- **指摘**: 重大度（block / warn / nit）、場所、改善案
- **仕様疑義**: 品質レビュー中に仕様矛盾を見つけた場合は第1段へ差し戻しと明記（仕様判断でごまかさない）
