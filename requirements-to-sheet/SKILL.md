# 要件定義をスプレッドシートに出力する

開発計画や要件定義の内容を、**要件定義フォーマット**に沿って表形式に整理し、Google スプレッドシートに書き込む。

## いつこのスキルを使うか

- 「要件定義をスプレッドシートに出力してほしい」と言われたとき
- 開発計画の要件・タスク・リスクなどをスプレッドシートで一覧にしたいとき
- 議事録や仕様を「要件定義のフォーマット」に落としてシートに書きたいとき

## 前提

- **gws** または **Sheets API** でスプレッドシートに書き込めること。gws の場合は `gws auth login -s sheets` 済み。
- 書き込み先スプレッドシートの **ID** と **シート名** をユーザーが指定できること。

## 手順（エージェントが行うこと）

1. **ソースを読む**
   対象の開発計画または要件定義の Markdown を読み、概要・目的・フェーズ・タスク・リスク・要確認事項などを把握する。

2. **要件定義フォーマットに変換する**
   [references/requirements-format.md](references/requirements-format.md) に従い、次のような「行データ」を組み立てる。
   - **シート「概要」**: 目的・スコープ・参照仕様を 1 行〜数行の見出し＋内容で構成。
   - **シート「要件一覧」**: 列は `セクション | ID | 要件（内容） | 備考・依存`。業務要件（B-xx）、機能要件（F-xx）、非機能要件（NF-xx）、タスク（Phase x の No.）、リスク・要確認 を同じ表形式で並べる。
   - **シート「フェーズ・マイルストーン」**（任意）: フェーズ名・内容・成果物・想定時期。

   開発計画に「要件定義」セクションが明示されていない場合は、タスク一覧・リスク・前提条件から要件らしい行を抽出し、ID は `T-1`, `T-2`（タスク）や `R-1`（リスク）など一意になるよう付与する。

3. **スプレッドシートに書き込む**
   - 組み立てた行データを **2 次元配列** `values`（先頭行はヘッダー）にする。
   - gws の `values update` または Sheets API で書き込む。
   - 複数シートに分ける場合は、シートごとに `range` を変えて update を実行する。

## 出力例（要件一覧の列イメージ）

| セクション   | ID   | 要件（内容）                                           | 備考・依存   |
|-------------|------|--------------------------------------------------------|--------------|
| 業務要件    | B-01 | 既存フォームを新フォーム1本に統合する                   |              |
| 機能要件    | F-01 | フォームは21項目を実装する                             | 仕様書4節    |
| タスク      | 1-1  | Supabase テーブル定義                                   |              |
| リスク・要確認 | R-1  | 文言が未確定のため仮文言で実装               | 確定後に差し替え |

## 注意

- **gws の `--params`** は **camelCase** の JSON 文字列で渡す。
- スプレッドシートの **シートが存在しない** 場合は、先に API でシートを追加するか、ユーザーに「指定シートを手動で作成してください」と案内する。

## Troubleshooting

### エラー: gws で読めない・書けない

**原因**: スコープ、ファイル ID 誤り、共有設定。

**対処**: **gws-drive** / **gws-sheets-*** と **gws-params-encoding** を確認し、ID と range を再確認する。

### エラー: Sheets API（Node 等）で更新できない

**原因**: 認証、シート名・range の誤り、権限。

**対処**: **sheet-api-update** の前提を確認し、API の有効化と認証情報を見直す。
