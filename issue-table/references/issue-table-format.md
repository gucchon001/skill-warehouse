# 課題管理表フォーマット（スプレッドシート用）

課題・タスク・バグ・要確認事項を一覧で管理するスプレッドシートの**列定義**と**推奨値**。

## 列定義

| 列名       | 説明 | 例・推奨値 |
|------------|------|------------|
| **No**     | 行番号または課題番号。並び替え・参照用。 | 1, 2, 3 または K-001 |
| **課題名** | 課題・タスクのタイトル。簡潔に。 | 「ログイン画面のバリデーション追加」 |
| **種別**   | 課題の種類。 | バグ / 機能 / タスク / 要確認 / 改善 |
| **優先度** | 重要度・緊急度。 | 高 / 中 / 低 または P0 / P1 / P2 |
| **状態**   | 進捗状態。 | 未着手 / 対応中 / レビュー中 / 完了 / 保留 |
| **担当**   | 担当者（名前またはイニシャル）。 | 山田 / 未定 |
| **期限**   | 目標日または期日。 | 2025-03-20 / 未定 |
| **備考**   | 依存・参照・補足。 | 要件 No.3 と連動、要確認: 原口 |

## シート構成

- **1 シートで完結**が基本。シート名は「課題管理表」「課題一覧」「Issues」など任意。
- 先頭行は**ヘッダー**。2 行目以降がデータ行。
- フィルタや並び替えを使う場合は、ヘッダー行を固定する。

## values の形式（gws / API 書き込み用）

- 2 次元配列。先頭行はヘッダー。
- 例: `[ ["No","課題名","種別","優先度","状態","担当","期限","備考"], ["1","...","機能","高","未着手","","",""] ]`
- 空の表を作るだけならヘッダー行のみでよい。

## 開発計画・PRD との連携

- 要件定義の「リスク・要確認」→ 種別「要確認」で課題管理表に 1 行追加できる。
- タスク（Phase 1 の No. など）→ 種別「タスク」で課題管理表に展開できる。
