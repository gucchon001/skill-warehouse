# 課題管理表をスプレッドシートで作る

課題・タスク・バグ・要確認事項を一覧にする**課題管理表**の構成を定義し、指定した Google スプレッドシートに書き込む。初回だけ `gws auth login -s sheets` を実行すること。

## いつこのスキルを使うか

- 「課題管理表を作って」「課題一覧をスプレッドシートに出力して」と言われたとき
- 開発計画のタスク・リスクを課題管理表形式でシートに展開したいとき
- 新規シートに課題管理のテンプレート（ヘッダー＋列定義）を用意したいとき

## 前提

- 書き込み先スプレッドシートの **ID** と、書き込む **シート名**（または gid）をユーザーが指定できること。

## 手順（エージェントが行うこと）

1. **列定義を確認する**
   [references/issue-table-format.md](references/issue-table-format.md) に従い、ヘッダー行と列を決める。標準は `No | 課題名 | 種別 | 優先度 | 状態 | 担当 | 期限 | 備考`。

2. **スプレッドシートに書き込む**
   - ヘッダー行（＋必要なら空のデータ行）を用意し、gws または Sheets API で書き込む。
   - 既存シートを上書きする場合は、先に範囲を確認してから上書きする範囲を決める。ラベル列を値で上書きしないよう注意する。

## 出力例（ヘッダーのみ）

| No | 課題名 | 種別 | 優先度 | 状態 | 担当 | 期限 | 備考 |

データ行は空でよい。ユーザーが手動で追記する。

## 参照

- 列定義・推奨値: [references/issue-table-format.md](references/issue-table-format.md)
- gws の `--params` に JSON を渡す実装では [gws-params-encoding](../gws-params-encoding/SKILL.md) を参照する。

## Troubleshooting

### エラー: gws で読めない・書けない

**原因**: スコープ、ファイル ID 誤り、共有設定。

**対処**: **gws-drive** / **gws-sheets-*** と **gws-params-encoding** を確認し、ID と range を再確認する。

### エラー: Sheets API（Node 等）で更新できない

**原因**: 認証、シート名・range の誤り、権限。

**対処**: **sheet-api-update** の前提を確認し、API の有効化と認証情報を見直す。
