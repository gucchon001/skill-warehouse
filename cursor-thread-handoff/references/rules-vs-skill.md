# 常時ルール（Rules）と本スキルの役割分担

ホストごとの Rules の**置き場所と拡張子**は **`skill-builder` の `references/skill-folder-spec.md` §9.2** を正とする。

## スキル（`thread-handoff`）

- **可変の文脈**: いつ・何を `HANDOFF` に書くか、カットオフの判断、テンプレの使い方
- **トリガー時に Read** される想定（Progressive Disclosure）

## 常時ルール（各ホスト）

- **不変に近いプロジェクト方針**（テスト方針、禁止事項、スタイル）
- 引き継ぎと組み合わせるなら **例**（短いだけ）:

```markdown
ユーザーが `HANDOFF.md` を会話のコンテキストに含めたときは、その内容を当該タスクの正とし、会話の推測で上書きしない。
```

（Cursor の `@`、Claude Code の file reference 等、製品の「ファイルを文脈に載せる」操作に読み替える。）

## 置いてはいけないもの

- スレッドごとに変わる長文のハンドオフ全文
- API キーやトークン（ルール・スキル・HANDOFF のいずれにも平文で書かない）
