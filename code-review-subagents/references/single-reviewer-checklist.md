# 単一レビュアー用チェックリスト（パターン A）

サブエージェント `code-reviewer`（1 体）向け。小さめの PR や仕様が既に固定されている変更で、2 段ゲートまで要らないときに使う。

## 入力

- 差分・対象ファイル、要件または Issue の参照（無ければ NEEDS_INFO で質問）。

## 一度に見る項目（優先順）

1. 仕様・スコープ: 依頼範囲から外れていないか。
2. 正しさ: ロジックバグ、エッジケース、エラー処理。
3. 品質: 可読性、命名、既存パターンとの一貫性。
4. テスト: 変更に見合うテストの有無。
5. セキュリティの目視: 明らかな注入・漏洩・認可漏れ（詳細監査は別タスク）。

## 出力

- 判定: PASS / FAIL / NEEDS_INFO
- 指摘: 重大度（block / warn / nit）、場所、修正案（読み取りレビューなら提案のみでよい）

仕様と品質を厳密に分離したゲートが必要なら automated-2stage-review（パターン B）を使う。
