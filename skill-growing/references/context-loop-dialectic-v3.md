# スキルを育てるスキルとコンテキストループ — 10回弁証法 v3（GPT-5.4 独立思考版）

**目的**: v1・v2の出力を一切参照せず、完全に独立した思考でスキル性能の本質とコンテキストループの回し方を弁証法で導出する。

---

## ラウンド 1：「スキルの性能」とは何を測っているのか

| | |
|---|---|
| **テーゼ** | スキルの性能とは「エージェントがタスクを正しく完了する確率」であり、スキルの内容が正確であるほど性能が高い。 |
| **クリティカルな問い** | 正確なスキルが存在しても、それがretrievalで選ばれなければ？選ばれても読まれなければ？読まれても従われなければ？ |
| **アンチテーゼ** | スキルの性能は単一の指標ではなく、パイプライン全体の歩留まりで決まる。selection → injection → adherence → execution。 |
| **合** | スキルの性能 = selection × injection × adherence × execution の連鎖確率。育てるとは4段階すべてのボトルネックを解消すること。 |

## ラウンド 2：selection の本質

| | |
|---|---|
| **テーゼ** | descriptionにトリガーワードを多く含めれば選ばれやすくなる。 |
| **クリティカルな問い** | descriptionが長く多義的だと、embeddingが汎化し特定の意図とのcos類似度が中途半端にならないか？ |
| **アンチテーゼ** | 広いdescriptionは「どこにでもいるがどこにも近くない」ベクトルを作る。 |
| **合** | selectionの最適化は「尖ったembedding」を作ること。少数の明確なトリガーに絞る。 |

## ラウンド 3：injection の本質

| | |
|---|---|
| **テーゼ** | 選ばれたら全文が載るのでinjectionは問題にならない。 |
| **クリティカルな問い** | スキルAがコンテキストを占有すると、スキルBや会話履歴やコードが押し出されるのでは？ |
| **アンチテーゼ** | スキル自身の情報量が増えるほどシステム全体の情報量が減るパラドクス。 |
| **合** | injectionの最適化は「最小の面積で最大の指示」= 符号化効率。Progressive disclosureは遅延評価。 |

## ラウンド 4：adherence の本質

| | |
|---|---|
| **テーゼ** | コンテキストに載った指示にはLLMは従う。 |
| **クリティカルな問い** | post-trainingで学習したデフォルト行動パターンとスキルの指示が衝突したとき、どちらが勝つか？ |
| **アンチテーゼ** | 曖昧な指示や弱い表現はデフォルトに負ける。指示とデフォルトの綱引きで不安定な出力になる。 |
| **合** | LLMのデフォルトで足りることは書かない。デフォルトに反することだけ強い制約語で書く。 |

## ラウンド 5：execution の本質

| | |
|---|---|
| **テーゼ** | 指示に従えば実行は成功する。失敗は環境の問題。 |
| **クリティカルな問い** | スキルは書かれた時点の世界のスナップショット。環境は変わるがスキルは変わらない。 |
| **アンチテーゼ** | 時間が経つほどスナップショットと現実のドリフトが蓄積する。 |
| **合** | executionの維持は「ドリフト検知」にかかっている。検証スクリプト = ドリフト量の測定装置。 |

## ラウンド 6：どの段階が最もボトルネックか

| | |
|---|---|
| **テーゼ** | 4段階は均等に重要。 |
| **クリティカルな問い** | どの段階の失敗が最もサイレントか？ |
| **アンチテーゼ** | adherenceの失敗は「なんか微妙な結果」として現れるだけで原因特定が極めて困難。最もサイレント。 |
| **合** | adherenceがスキル性能の最大のサイレントボトルネック。 |

## ラウンド 7：adherenceを高める「表現」

| | |
|---|---|
| **テーゼ** | 命令形で強く書けば従う。 |
| **クリティカルな問い** | 命令が10個並んだとき、すべてに等しく従えるか？ |
| **アンチテーゼ** | 制約が増えるほど各制約へのadherenceが下がる。10の命令を全部守るより、3つを完璧に守る方が実質的な性能は高い。 |
| **合** | 制約は3つに絞れ。「書かないこと」がadherenceを上げる。 |

## ラウンド 8：ループは何を学習しているのか

| | |
|---|---|
| **テーゼ** | コンテキストループはスキルの「内容」を改善するプロセス。 |
| **クリティカルな問い** | 内容を変えるだけでは4段階のうちexecutionしか改善しないのでは？ |
| **アンチテーゼ** | 現在のループはexecutionに偏重している。 |
| **合** | ループは4軸（selection, injection, adherence, execution）すべてを学習対象にすべき。 |

## ラウンド 9：ループの「収束」と「発散」

| | |
|---|---|
| **テーゼ** | ループを回すほどスキルは成熟に収束する。 |
| **クリティカルな問い** | 追記でexecution改善 → 肥大化でinjection劣化 → 圧縮でexecution劣化の振動が起きないか？ |
| **アンチテーゼ** | 追記型ループは振動する。収束しない。 |
| **合** | 「追記」ではなく「置換」を基本操作にする。総量は一定か減少。スキルは見つけた時より短くして返す。 |

## ラウンド 10：この議論をスキルにどう反映するか

| | |
|---|---|

| **クリティカルな問い** | R7で「制約は3つに絞れ」と結論したのに、10個の洞察を全部書いたら自己矛盾では？ |
| **アンチテーゼ** | 自己矛盾する。 |

## 最終結論

### 性能を最も落とす本質

1. **Adherenceのサイレントな劣化** — 書く量が増えるほど各制約への追従確率が下がる。スキルは書くほど弱くなる。
2. **スナップショットと現実のドリフト** — 検証なしではドリフトに気づかない。
3. **追記型ループの振動** — 追記を基本操作にしている限りループは収束しない。

### 性能を最も引っ張る本質

1. **制約の「引き算」** — 本当に重要な3つだけを残す。「書かないこと」がadherenceを上げる。
2. **4段階パイプライン全体の歩留まり向上** — selection, injection, adherence, executionすべてを育てる。
3. **置換型ループ** — 追記ではなく置換。総量は一定か減少。

### コンテキストループの回し方（4軸診断ループ）

| ステップ | 問い | 操作 |
|----------|------|------|
| 1. Selection診断 | 正しい場面で選ばれているか？ | description/トリガー調整 |
| 2. Injection診断 | 他の文脈を圧迫していないか？ | 冗長部分の圧縮・参照分離 |
| 3. Adherence診断 | 書いてあることに従っているか？制約は3つ以内か？ | 弱い制約の削除、表現強化 |
| 4. Execution診断 | 手順は現在の環境で動くか？ | 検証実行、手順の置換 |

### 最も破壊力のある1つの原則

> **スキルの性能は、書いてあることの「量」ではなく、書いてあることに「従われる確率」で決まる。**
