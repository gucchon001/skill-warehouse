# 設定値一覧の書き方

選択肢（コード）をテーブル定義書・フォーム・API で一貫して扱うための設定値一覧の作り方。スキル「table-definition-database-design」から参照する。

---

## 1. 目的

- フォームの選択肢・API のバリデーション・DB に保存する値で**同じコード**を使う。
- 変更時に一箇所（定義書またはマスタ）を更新すれば、実装の参照元が分かるようにする。

---

## 2. 一覧表の列

| 列名 | 内容 | 例 |
|------|------|-----|
| **コード** | 保存する値（API・DB）。文字列または数値のコード | `1`, `2`, `direct`, `line` |
| **表示ラベル** | UI に表示する文言 | `1. 東京、神奈川、埼玉、千葉` |
| **説明** | 業務上の意味・備考（任意） | 分岐で「1,2,3のいずれか」に使う |

---

## 3. DB での扱い

- コードは **TEXT** で保存する場合が多い（PostgreSQL）。
- 許容値を限定する方法:
  - **CHECK 制約**: `CHECK (col IN ('1','2','3','4'))`
  - **参照テーブル**: マスタテーブルを用意し、FK で参照する（将来の追加・多言語化に向く）
- 複数選択の場合は、カンマ区切り `'1,2,3'` や配列型で保存。定義書に「複数選択可。カンマ区切りで保存」などと書く。

---

## 4. フォーム・API との対応

- フォームの `<option value="1">` の `value` は、定義書の**コード**と一致させる。
- API のバリデーションスキーマでは定義書のコードを列挙する。
- 定義書を更新したら、form-options・schema・DB の CHECK をまとめて更新する。

---

## 5. 例（希望勤務地）

**設定値一覧（定義書内の表）**

| コード | 表示ラベル | 説明 |
|--------|------------|------|
| 1 | 1. 東京、神奈川、埼玉、千葉 | 分岐で「1・2・3のいずれか」に含む |
| 2 | 2. 大阪、兵庫、京都 | 同上 |
| 3 | 3. 愛知 | 同上 |
| 4 | 4. その他 | 「その他のみ」の判定に使用 |

**DB**: `desired_work_location TEXT`（複数選択のためカンマ区切りで保存）。許容文字は `'1','2','3','4'` のいずれかのみ。

**フォーム**: チェックボックスで value に `1`〜`4` を設定。送信時は配列 `['1','2']` を API に送り、API で `join(',')` して DB に保存する。
