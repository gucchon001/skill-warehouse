# UI・フロントエンドパターン（軽量レビュー）

> **新規UI作成・スタイリング全般は `frontend-design` を使う。**  
> 本スキルは既存UIを素早くチェックする場合のみ。

## When to Use

- 「ちょっとチェックして」「見やすくして」「コンポーネントを揃えて」
- すでに実装済みのUIに対して品質確認するとき
- DESIGN.md もなく、全体設計は不要なとき

→ 新規作成・DESIGN.md管理・美的方向性 → **`frontend-design`**  
→ ダッシュボードのファーストビュー・情報順序 → **`dashboard-first-view-ux`**

---

## 5観点チェックリスト

- [ ] **コントラスト**: 文字と背景のコントラスト比が WCAG AA (4.5:1) 以上か
- [ ] **余白**: セクション・カード間の余白でグルーピングが分かるか
- [ ] **見出し階層**: h1/h2/h3 のサイズ・ウェイトが一貫しているか
- [ ] **主要 CTA**: メインのリンク・ボタンが accent で強調されているか
- [ ] **状態**: 無効・読み込み中・エラーなどの状態が視覚的に区別できるか

### 追加確認（必要に応じて）

- [ ] モバイル: 狭い幅での折り返し・タップ領域 (≥44px) は十分か
- [ ] フォーカス: キーボード操作でフォーカスリングが見えるか
- [ ] アイコン単体: alt テキストまたは aria-label があるか

---

## コンポーネントと一貫性の原則

- **名前・役割・状態を揃える**: カード・バッジ・ボタンなど、プロジェクト内で名前と役割を統一する
- **主要 CTA を accent で**: 主要アクションは accent 色で一貫して強調、二次は控えめに
- **スキャナブル**: 見出し・カード・リストでブロックを分け、縦に流し読みできるようにする

## Troubleshooting

### 修正範囲が広くなり手に負えない

**原因**: UIの大半がデザインシステム未定義で、個別修正が場当たりになる。

**対処**: スコープを超えている。`frontend-design` で DESIGN.md から設計し直す。

### コントラストが通らないが色を変えたくない

**原因**: ブランドカラーがアクセシビリティ基準を満たさない。

**対処**: 文字色または背景色を微調整（明度±10%程度）。ブランドカラーは「ロゴ・装飾」に限定し、テキスト背景には使わない。
