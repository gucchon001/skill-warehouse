# テーブル定義書・データベース設計

「テーブル定義書を書いて」「DB設計して」「テーブル定義書を作成したうえでフォームに落としたい」と言われたとき、または新規テーブル・スキーマ変更の設計を行うときに使う。

---

## When to Use

- 新規テーブルを追加する前に、定義書で型・制約・設定値を決めたいとき
- 既存の単一テーブルを複数テーブルに分割する設計をするとき
- 氏名の姓・名分離や他システムとの連携方針を定義に含めたいとき
- 選択肢（コード）の設定値一覧をドキュメント化し、フォーム・API・DB で一貫させたいとき

---

## 設計の流れ

1. **要件の把握**
   フォーム仕様・業務フロー・既存テーブルとの対応を確認する。差分整理ドキュメントがある場合は読み、連携方針を定義書に書く。

2. **エンティティ・テーブル分割の決定**
   1 テーブルに全項目を並べるのではなく、役割ごとに分割する（例: 申込ヘッダ・個人情報・希望条件）。主キー・外部キー・1 対 1 / 1 対多の関係を決める。

3. **テーブル定義書の作成**
   次の内容をドキュメントに書く。
   - **論理テーブル一覧**: テーブル名・説明・主キー・他テーブルとの関係
   - **物理定義（各テーブル）**: カラム名・**データ型（TEXT / VARCHAR(n) / DATE / INTEGER / UUID / TIMESTAMPTZ 等）**・NULL/NOT NULL・CHECK・DEFAULT・COMMENT。外部キー・一意制約・インデックス
   - **氏名・複合項目の方針**: 氏名を「姓・名・姓カナ・名カナ」で持つか、フルネーム 1 つのみか。他システムが姓・名を要求する場合の連携方針を明記する
   - **設定値一覧**: 選択肢項目ごとに「コード・表示ラベル・説明」の表を用意する。DB ではコードを TEXT で保存し、CHECK または参照で許容値を限定する旨を定義書に書く

4. **設定値一覧の配置**
   定義書内の表で完結させるか、別ファイル（CSV / MD）で参照するか決める。フォームの選択肢と定義書のコードを一致させる。

5. **マイグレーション・実装**
   定義書に沿って SQL マイグレーションを作成する。適用後に型生成を実行する。

---

## テーブル定義書に含める項目（チェックリスト）

- [ ] エンティティ概要・テーブル一覧（役割・関係）
- [ ] 各テーブルのカラム: 名前・**型**・NULL/NOT NULL・CHECK・DEFAULT・COMMENT
- [ ] 主キー・外部キー・一意制約・インデックス
- [ ] 氏名の扱い（姓・名分離の有無と、他システム連携方針）
- [ ] 設定値一覧（コード・ラベル・説明）。コードで保存する場合の許容値（CHECK または参照）
- [ ] RLS 方針（必要な場合）

詳細なチェック項目は `references/table-definition-checklist.md` を参照。

---

## 設定値一覧の書き方

- フォームの選択肢・API のバリデーション・DB の CHECK で**同じコード**を使う。
- 表の列: **コード**（保存値）・**表示ラベル**（UI 表示）・**説明**（任意）。
- 例: 希望勤務地 → コード `1`〜`4`、ラベル「1. 東京、神奈川、埼玉、千葉」等。DB は `TEXT` で `CHECK (desired_work_location_code IN ('1','2','3','4'))` または複数選択の場合はカンマ区切りで保存し、許容文字を定義書に書く。

詳細は `references/setting-values-catalog.md` を参照。

---

## 注意

- 定義書を先に確定してからマイグレーションとフォーム実装に進む。実装と定義書の齟齬が生じたら定義書を更新するか実装を合わせるかを明示的に決める。
- 既存テーブルを破棄する場合は、本番・ステージングにデータがあるか確認し、必要なら移行手順を別途書く。

## Troubleshooting

### エラー: gws で読めない・書けない

**原因**: スコープ、ファイル ID 誤り、共有設定。

**対処**: **gws-drive** / **gws-sheets-*** と **gws-params-encoding** を確認し、ID と range を再確認する。

### エラー: Sheets API（Node 等）で更新できない

**原因**: 認証、シート名・range の誤り、権限。

**対処**: **sheet-api-update** の前提を確認し、API の有効化と認証情報を見直す。
