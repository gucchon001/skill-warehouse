# 要件定義フォーマット（スプレッドシート用）

開発計画・要件定義の Markdown をスプレッドシートに出力する際の、**シート構成**と**列定義**。

## シート構成

| シート名         | 用途 |
|------------------|------|
| **概要**         | 目的・スコープ・参照仕様を短い見出し＋内容で記載。見やすさのため 2 列（項目／内容）がよい。 |
| **要件一覧**     | 業務要件・機能要件・非機能要件・タスク・リスク・要確認を 1 つの表で管理。 |
| **フェーズ・マイルストーン**（任意） | フェーズ名・内容・成果物・想定時期。 |

## 要件一覧の列定義

| 列名           | 説明 | 例 |
|----------------|------|-----|
| **セクション** | 要件の種別。同じ値でグルーピングする。 | 業務要件 / 機能要件 / 非機能要件 / タスク(Phase1) / リスク・要確認 |
| **ID**         | 一意の識別子。並び替え・参照用。 | B-01, F-01, NF-01, 1-1, 2-1, R-1 |
| **要件（内容）** | 要件またはタスクの本文。簡潔に。 | 「フォームは21項目を実装する」 |
| **備考・依存** | 依存タスク・参照仕様・確認先・未決定時のメモ。 | 1-2, 仕様書4節 |

## 開発計画から行へ落とし込むときの対応

- **目的・スコープ** → 「概要」シートの行、または「要件一覧」のセクション「目的」で 1〜2 行。
- **タスク一覧** → セクション「タスク(Phase1)」等、ID は `1-1`, `1-2` 形式。
- **リスク・前提条件** → セクション「リスク・要確認」、ID は `R-1`, `R-2` など。
- **未決定・要確認事項** → 同様に「リスク・要確認」または「要確認」、確認先を備考に記載。
- 既存の要件定義 MD に B-xx, F-xx がある場合は、その ID をそのまま使う。

## 概要シートの列（2 列推奨）

| 項目       | 内容 |
|------------|------|
| タイトル   | プロジェクト名 |
| 目的       | プロジェクトの目的 |
| スコープ   | 対象範囲 |
| 参照仕様   | 仕様書・要件定義・議事録のパスまたは URL |

## values の形式（gws / API 書き込み用）

- いずれのシートも **先頭行はヘッダー**、2 行目以降がデータ。
- `values` は 2 次元配列: `[ ["セクション","ID","要件（内容）","備考・依存"], ["業務要件","B-01","...",""], ... ]`
- 概要シート: `[ ["項目","内容"], ["目的","..."], ["スコープ","..."] ]`
