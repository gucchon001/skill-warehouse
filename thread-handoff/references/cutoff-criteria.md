# カットオフ（スレッド／セッションを切るタイミング）

製品に**公式のトークン閾値が公開されていない**ことが多いため、**運用基準**として使う。エージェントはユーザーの文脈に合わせて取捨選択する。

## 推奨するタイミング

- 大きなタスク単位が終わった
- 会話の主題が変わった（別 Issue・別ルート・別仕様）
- 同じファイル・同じエラーの説明を繰り返している
- ユーザーが「長い」「切りたい」と言った
- **主トリガー**: 「**新しいチャットする準備**」（および「新しくチャットを切り替える準備」等の同義）

## 新セッション側（受け手エージェント）

- ユーザーがコンテキストに載せた正本を Read し、推測で上書きしない
- `最終更新` が古ければリポジトリの現状と照合し、ズレを明示する
