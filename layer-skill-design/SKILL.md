# レイヤー型スキル設計

複数スキルとその参照データを、「考える」「作る」「参照する」に分離し、フォルダ構成を統一するときの指針。

## いつ使うか

- ユーザーが「スキルをレイヤーで整理して」「フォルダ構成をレイヤー型にして」と依頼したとき
- プロンプトや設定をスキル間で共有するフォルダ構成を決めるとき
- 新規スキルを既存のレイヤー構成に追加するとき

## レイヤー構成

### 思考レイヤー（考える）

調査・企画・評価・最適化など、タスクの「思考」フェーズを担当するスキル。

| スキル例 | 役割 |
|----------|------|
| research | 調査・情報収集 |
| planning | 企画・構成設計 |
| review | 評価・改善提案 |
| optimize | プロンプト最適化 |

### 制作レイヤー（作る）

執筆・整形・スライド・画像プロンプトなど、アウトプットを「制作」するスキル。

| スキル例 | 役割 |
|----------|------|
| writer | 長文執筆 |
| sns-formatter | SNS 投稿文整形（X 形式など） |
| slide-composer | スライド構成 / HTML |
| visual-prompt | 画像生成プロンプト作成 |

### データレイヤー（参照する）

スキルが参照するプロンプト・テンプレート・設定を格納する。次のフォルダ構成に従う。

## データレイヤー：フォルダ構成

スキル用の参照データは、次のツリーで配置すること。

```
prompt/
  スライド用デザインスタイル/    # スライドのデザイン・スタイル用プロンプト
  Web_UI/
    テンプレート/                # Web UI テンプレート用プロンプト
  画像/
    図解/                       # 図解用プロンプト
    サムネイル/                 # サムネイル用プロンプト
os/                             # identity, protocol, preferences 等のシステム・環境設定
```

- **prompt/** の下は用途別にディレクトリを分け、必要に応じてサブフォルダを追加する。
- **os/** には識別子・プロトコル・ユーザー設定など、スキル共通の設定を置く。

## 運用ルール

1. 新規スキルを追加するときは、思考／制作のどちらのレイヤーに属するかを決める。
2. プロンプトやテンプレートを追加するときは、上記の **データレイヤー** のどこに置くか決めてから配置する。
3. 既存のスキル群をレイヤー型に移行するときは、まずデータレイヤーの `prompt/` と `os/` を作成し、既存のプロンプト・設定を移す。

## 配置先

- レイヤー構成を**このプロジェクトだけ**に適用する場合: `.cursor/skills/` 配下の各スキルと、プロジェクトルートに `prompt/` および `os/` を用意する。
- **全プロジェクトで共通**にしたい場合: グローバルスキル（`~/.cursor/skills/`）と、ユーザー設定ディレクトリなどに `prompt/`・`os/` を用意する。

## 関連スキル

- 個別の **SKILL.md** の作成・編集は **skill-builder**（スキル構築スキル）を参照する。レイヤー分けやデータレイヤーのフォルダ構成は本スキル（layer-skill-design）に従う。

## Troubleshooting

### エラー: スキルが読み込まれない・参照先が無い

**原因**: `~/.cursor/skills/<name>/` のフォルダ名と `SKILL.md` の `name` が不一致、または `references/` の相対パスが誤り。

**対処**: フォルダ名と YAML の `name` を同一 kebab-case に揃える。`references/` へのリンクをスキル根からの相対パスで確認する。

### エラー: 手順どおりに反映されない

**原因**: Cursor のバージョン差、または別ホスト（Claude Code / Antigravity）ではスキル配置パスが異なる。

**対処**: **skill-folder-spec.md** §9 でホスト別パスを確認する（**skill-builder** 内の `references/skill-folder-spec.md`）。
