# WBS シートの構成（サンプル準拠）

サンプルスプレッドシートを参考にした WBS シートの列定義。プロジェクトにより列の増減は可。

---

## 見出し・メタ行

| 行の例 | 意味 |
|--------|------|
| プロジェクトWBS | タイトル |
| 更新日 | シート最終更新日 |
| 起点日 | スケジュールの起点日 |
| PJフォルダ | プロジェクト名・フォルダパス等 |

---

## データ行の列（ヘッダー）

| 列 | 項目名 | 説明 |
|----|--------|------|
| A | NO. | 行番号・タスク番号（例: 1, 2, 3 または 1-1, 2-1） |
| B | 大項目 | フェーズ・大きな成果物（例: Phase 1: 基盤） |
| C | 中項目 | 大項目の下位（例: DB・API・分岐ロジック） |
| D | 小項目 | 中項目のさらに分解（必要に応じて） |
| E | タスク | 実行単位のタスク名 |
| F | ステータス | 未着手 / 実行中 / 完了 等 |
| G | オーナー | 担当者 |
| H | 関係者 | 関係するメンバー・役割 |
| I | 開始日 | 開始予定日 |
| J | 終了日 | 終了予定日 |
| K | 資料 | リンク・参照ドキュメント |
| L | 備考 | 前提・制約・リスクメモ |

その右側に日付列を並べてガントチャート風にすることも可。

---

## 階層の対応

- **大項目** → **中項目** → **小項目** → **タスク** の順で階層的に分解する。
- 成果物を出発点にし、作業（タスク）まで落とす。MECE・スコープを守る。
- 「肝タスク」は Step 3 で洗い出した対策タスク。WBS 上ではタスク列に明示する。

---

## gws / API で書き込む場合

- シート名はスプレッドシートの `title` を使う（gid は使わない）。
- ラベル行（ヘッダー）と値行を混同しない。
- 範囲は「シート名!A1:Lnn」のようにヘッダー含めて指定するとずれを防げる。
