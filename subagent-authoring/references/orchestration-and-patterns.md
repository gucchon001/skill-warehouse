# サブエージェントの運用パターンと設計のコツ

親スキル: **subagent-authoring** の `SKILL.md`。

サブエージェントは親から委譲され、**独立したコンテキスト**を持つことが多い。複雑タスクの並列化・専門化に向く。以下は **ホスト横断で有効な設計思想**である。API 名（例: fork 相当の指定）は **製品・バージョンで異なる**ため、実装前に公式ドキュメントを確認する。

---

## 1. 主な設計パターン

### 1.1 分離型（隔離実行）

親の会話に大量の読み取りログや中間成果を載せず、**サブ側で広いコンテキストを使い、親には要約・結論だけ返す**パターン。

- **向いている例**: 大規模リサーチ、詳細コードレビュー、ログの総当たり調査。
- **ホストの能力**: 「fork」「subtask」「別ウィンドウ」など、**隔離された実行**を明示できる環境ではより効果的。**キー名・指定方法はホスト公式を確認**（このファイルでは固定構文を定めない）。

### 1.2 マルチエージェント協調型（チーム構成）

複数サブエージェントを **並列** または **順序付き** で起動し、一つのタスクを分割して解くパターン。

- **向いている例**: 調査 A・B を並列 → 統合役がマージ、実装 → レビュー → 修正のパイプライン。

## 2. 設計のコツ（実務）

### 2.1 1 エージェント ＝ 1 タスク（1 役割）

1 つのサブエージェントに「調査しつつ分析しつつ執筆」まで持たせると出力が散漫になりやすい。

- **リサーチ担当 / 統合担当 / 執筆担当** のように **役割を分離**し、それぞれ専用の `.md` 定義にする。

### 2.2 依存関係と遷移を明示する

複数サブを動かすとき、**Skill（親の手順書）の本文**に次を書く。

- **並列でよいか、順序必須か**（DAG のイメージで簡潔に）。
- **完了後に次に起動するサブエージェント（またはスキル）**を、箇条書きやマーカー（例: `NEXT: invoke subagent spec-reviewer`）で固定し、**推論による手順飛ばしを減らす**。

マルチステップの Skill を書くときは **skill-builder** の PD と整合させ、長い遷移表は Skill の `references/` に退避する。

### 2.3 プロンプトの分離（段階的開示）

実装用・仕様レビュー用・コード品質レビュー用など、**サブごとの指示を 1 巨大ファイルに詰め込まない**。

- **サブエージェント 1 体につき 1 `.md`** が基本。
- 共有したい長文は **`references/` やデータレイヤー**（**layer-skill-design**）に置き、各サブの本文から **必要なファイルだけ Read** させる。

### 2.4 段階的レビュー（ゲート・差し戻し）

完了宣言の前に **検証ゲート** を置く。

- 例: **仕様レビュアー**が合格するまで **コード品質レビュアー**を起動しない（仕様不一致のコードレビューは手戻りが増える）。
- Skill 本文に「フェーズ 1 → 合格条件 → フェーズ 2」と **順序を固定**する。

### 2.5 並列リサーチ ＋ 統合

同じ問いを **複数リサーチ用サブ**に並列で渡し、偏りを減らす。その後 **統合用サブ**が要約・矛盾解消・網羅性チェックを行う。

- 統合役は **入力が複数ソースであること**をプロンプトに明記する。

## 3. Skills（SKILL.md）との役割分担

| 層 | 役割の目安 |
|----|------------|
| **Agent Skill** | 「いつ・どの順で・どのサブ／ツールを使うか」の**オーケストレーション**、成果物の定義、ゲート条件。 |
| **サブエージェント `.md`** | 委譲された**単一役割**の振る舞い、出力形式、制約。 |

オーケストレーションを Skill に書くときは **subagent-authoring**（本ファイル）と **skill-growing**（既存 Skill の圧縮・分割）を併用する。

## Troubleshooting

### エラー: サブが並列で暴走し、親が制御できない

**原因**: 依存関係と「次の一手」が Skill に書かれていない。

**対処**: §2.2 に従い、親 Skill の本文または `references/workflow-*.md` に遷移を明示する。

### エラー: コンテキストがすぐ上限に達する

**原因**: 全サブのプロンプトやログを親に集約している。

**対処**: §1.1（分離）と §2.3（分離ファイル）を適用。要約だけ親に返す契約をサブの本文に書く。
