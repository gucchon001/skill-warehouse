# PJC チェックリスト・評価観点

チャーター作成・見直し時に、以下の観点で妥当性を確認する。

---

## 全体

| 観点 | チェック内容 |
|------|--------------|
| **プロジェクト組成の妥当性** | 投資対効果を考えているか。1人作業をプロジェクト風に仕立てていないか。 |
| **プロジェクト名称** | 実際に当PJCを正しく認識できる名称になっているか。 |
| **要素間の整合性** | 背景 → 目的・成功状態、目的・成功状態 → 内容・方法・予算 のつながりが取れているか。 |

---

## 背景

| 観点 | チェック内容 |
|------|--------------|
| **課題の明確化** | なぜこのプロジェクトを設定するのか、解決したい課題の背景は言語化されているか。 |
| **自分ごとの定義** | 上司から降りてきた案件であっても、自分の言葉で背景を定義し直せているか。 |
| **論理的つながり** | 現状（AsIs）とあるべき姿（ToBe）のギャップ（問題）とその原因（Cause）の仮説が含まれているか。 |

---

## 目的・成功状態

| 観点 | チェック内容 |
|------|--------------|
| **SMARTの法則** | 目標は以下の要素を満たしているか。 |
| | • **S**pecific — 具体的で明確か |
| | • **M**easurable — 成果を数値で測定できるか |
| | • **A**chievable — 達成可能で現実的か |
| | • **R**elated — 上位目標や概念と関連しているか |
| | • **T**ime-bound — 達成期限が明確か |
| **アウトプットイメージ** | 行動・成果物・関係者の状態がクリアにイメージできるか。 |

---

## スコープ・影響範囲

| 観点 | チェック内容 |
|------|--------------|
| **欲張りすぎていないか** | 今回のPJCで**扱う範囲**と**扱わない範囲**が明確か。 |
| **フェーズ分割** | 期間が長く終わりが見えにくい場合、フェーズを分割してスコープを限定できているか。 |

---

## 内容・方法

| 観点 | チェック内容 |
|------|--------------|
| **手段の目的化回避** | 「作業」自体が目的になっていないか。何の効果を期待するかが意識されているか。 |
| **整合性** | 背景（課題）を解決するための具体的な手段になっているか。 |
