# デフォルトの 3 視点プロンプト（実装前クリティック）

親スキル: **pre-implementation-critics**。ホストで 3 サブ並列が難しい場合は順次でもよいが、視点の独立性は保つ。

## Critic A: 正確性・根拠

設計・調査結果の主張は証拠に支えられているか。前提・引用・再現手順は十分か。推測を事実として書いていないか。

## Critic B: 網羅性・漏れ

受け入れ条件・エッジケース・非機能（性能・セキュリティ・運用）が抜けていないか。

## Critic C: 代替仮説・リスク

別解・より単純な実装・別根本原因（バグ調査時）はないか。採用案の最大リスクと緩和策。

## 各サブの出力形式

- 判定サマリ（1〜3 文）
- 指摘（箇条書き、重大度つき）
- 実装前に必須の追記・検証があれば明記
